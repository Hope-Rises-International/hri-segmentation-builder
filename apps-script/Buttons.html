HRI Segmentation Builder — Button Reference

Synced from SPEC.md v3.4.1 (2026-04-29). Edit the Buttons.html file in the build repo to update; deploy cycle = sync cycle.
Button / Action What It Does When To Use What Gets Written / Changed Reversible?
Refresh Universe Re-fetches qualified universe from BQ cache. Applies exclusions (per OFF toggles), then waterfall + suppressions + segment assignment. Campaign change, or SF data has changed since last pull. Browser state. Yes — re-click.
Run Projection Computes economics per segment from Historical Baseline. Writes one row per segment to Draft tab. After universe loaded; want to see projection. Draft tab. Overwrites prior contents. Yes — re-run.
Edit Scenario (per-segment toggles, % slider, Holdout %, target type) Browser-side what-if. Adjusts inclusion / quantity / per-segment Holdout % (v3.4, default 5, range 0–5; soft orange warning below 3). No new SF query. Iterating to hit target; tuning per-segment ROI measurement vs. mailable count trade-off. UI only until Save / Approve. Yes — toggle back / Reset Overrides.
Save Scenario Saves current scenario state to Draft tab. Iterating across sessions or sharing with Bill. Draft tab updated. Partial — overwrites prior Draft.
Approve Locks Draft as final. Copies to Segment Detail tab. Triggers Generate Mailing File. After sign-off. Segment Detail tab + link_to_segments populated. Difficult — re-run upserts on same Campaign + Segment.
Generate Mailing File Produces Print + Matchback CSVs to Drive output folder. Matchback contains mailed + holdouts only (no trim residue). Auto on Approve, or independent regeneration. Two timestamped CSVs in Drive. Yes — new files have new timestamp.
Load to Salesforce DEFERRED. Will write Campaign + Campaign_Segment__c + CampaignMember post-merge-purge. After Faircom returns merge / purge file (separate processor). SF Campaign / Segment / Member / Account NCOA updates. Idempotent upsert.
Download Print File Direct download link from Drive. Preparing transmission to Faircom. Browser download. N/A.
Download Matchback File Direct download link from Drive. Internal HRI use. Browser download. N/A.
Bundle additional campaigns (multi-select) Adds a second / third campaign to the run. Cohorts route by prefix (v3.3): A → housefile / Cornerstone / Sustainer; M → Mid-Level / Major Gift; N → in-house Major Donor (only used when the Major Donor In-House Tier 2 toggle is OFF). One Print + one Matchback file pair per campaign. When a single mailing date carries multiple appeal codes (e.g. A2651 housefile + M2651 mid-level). Multiple file pairs in Drive — one per selected campaign. Yes — uncheck and re-run.
Major Donor In-House (Tier 2 always-on toggle) Default ON — suppresses donors where Major_Donor_In_House__c = "In House" from any standard mailing. They're managed via portfolio reports, not DM. Toggle OFF only when running an in-house-only mailing — the validator then requires an N-prefix campaign in the selection so the in-house cohort routes to its own file. Default state for every campaign. Flip OFF only for the rare in-house-only blast. Tier 2 suppression entry in the audit log. When OFF, in-house donors land in the N-prefix file. Yes — toggle ON and re-run.
Nuclear mode (double-confirmed toggle) Forces all GROUP toggles ON, bypasses Tier 2/3 + recent-gift + freq cap + holdout. Tier 1 hard suppressions still apply. Cohort routing preserved. Cohort-prefix validation still runs. Auto-resets to OFF after a successful run. Emergency appeals or full-housefile blasts only — operator must type "NUCLEAR" to activate. Standard outputs + a separate nuclear_run_<campaign>_<TS>.log file in the Drive output folder. Per-run only — toggle resets after each Approve.
